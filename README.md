# AmoCRM — Виджет «Распределение сделок»

Автоматически распределяет входящие сделки и сделки, перемещающиеся по этапам воронки, между менеджерами по настраиваемым правилам.

---

## Возможности

| Функция | Описание |
|---|---|
| **Round Robin** | Поочерёдное распределение сделок между выбранными менеджерами |
| **По загруженности** | Сделка назначается менеджеру с наименьшим числом открытых сделок |
| **История контакта/компании** | Если у контакта уже есть ответственный из списка — сделка назначается ему |
| **Рабочее расписание** | Распределение только в рабочие часы менеджера |
| **Правила по воронке/этапу** | Разные наборы менеджеров для разных воронок и этапов |
| **Digital Pipeline** | Интеграция с цифровой воронкой AmoCRM — настройка прямо на этапе |

## Быстрый старт

### 1. Бэкенд

```bash
cd server
cp .env.example .env
# Заполните AMO_CLIENT_ID, AMO_CLIENT_SECRET, AMO_REDIRECT_URI, STORAGE_PATH
composer install
php -S 0.0.0.0:8080 -t public
```

### 2. OAuth

В настройках интеграции AmoCRM укажите Redirect URI:

```
https://dist.koagency.me/oauth/callback
```

При установке виджета AmoCRM перенаправит пользователя на этот адрес, и токены сохранятся автоматически.

### 3. Упаковка виджета

Создайте ZIP-архив содержащий файлы виджета (без папки `server/`):

```bash
zip -r deal-distribution-widget.zip manifest.json widget.js css/ i18n/ images/
```

Загрузите архив в раздел **Настройки → Интеграции → Виджеты** в AmoCRM.

### 4. Настройка

1. Откройте виджет в AmoCRM → **Установить**.
2. В настройках укажите **URL сервера** (адрес запущенного бэкенда).
3. Выберите **метод распределения** (Round Robin / По загруженности).
4. Добавьте **правила**: выберите воронку, этап и менеджеров.
5. Опционально: включите проверку истории и расписания.

## API бэкенда

### `POST /api/distribute`

Распределить сделку.

```json
{
  "account_id": "12345",
  "lead_id": 67890,
  "pipeline_id": 111,
  "stage_id": 222,
  "distribution_method": "round_robin",
  "rules": [
    {
      "pipeline_id": 111,
      "stage_id": 222,
      "managers": [{"id": 1}, {"id": 2}, {"id": 3}],
      "check_history": true,
      "check_schedule": false
    }
  ]
}
```

**Ответ:**
```json
{
  "status": "ok",
  "assigned_to": "Иван Иванов",
  "user_id": 2
}
```

### `PUT /api/settings`

Сохранить настройки виджета для аккаунта.

### `GET /api/settings`

Получить настройки виджета (заголовок `X-Account-Id`).

### `GET /oauth/callback`

OAuth 2.0 Callback — AmoCRM перенаправляет сюда после авторизации.

## Расписание менеджеров

Создайте файл `STORAGE_PATH/schedules/{accountId}/{userId}.json`:

```json
{
  "timezone": "Europe/Moscow",
  "days": {
    "mon": {"start": "09:00", "end": "18:00"},
    "tue": {"start": "09:00", "end": "18:00"},
    "wed": {"start": "09:00", "end": "18:00"},
    "thu": {"start": "09:00", "end": "18:00"},
    "fri": {"start": "09:00", "end": "18:00"},
    "sat": null,
    "sun": null
  }
}
```

`null` означает выходной день.
