# Basa Finance

Веб-сервис учёта финансов для команды: доходы и расходы, счета в разных
валютах, контрагенты и проекты, долги (дебиторка/кредиторка), бюджеты и
аналитика. Гибкие роли и права доступа на уровне базы (RLS).

## Стек

- **Next.js 14** (App Router, TypeScript) + Tailwind CSS
- **Supabase** — Postgres, Auth, Row Level Security
- **Vercel** — хостинг фронтенда

## Локальный запуск

```bash
cd finance-app
cp .env.local.example .env.local   # ключи уже подставлены
npm install
npm run dev
# http://localhost:3000
```

## Структура

```
finance-app/
├── supabase/migrations/   # SQL-схема (источник истины по БД)
│   ├── 0001_foundation.sql       # профили, команды, роли, RLS
│   ├── 0002_finance_core.sql     # счета, операции, долги, бюджеты…
│   └── 0003_harden_functions.sql # ужесточение доступа к функциям
└── src/
    ├── app/
    │   ├── login/                # вход / регистрация / magic-link
    │   ├── auth/callback/        # обработчик авторизации
    │   └── (app)/                # защищённая зона: дашборд, команда и др.
    ├── components/               # UI-компоненты
    └── lib/
        ├── supabase/             # клиенты Supabase (browser/server/middleware)
        ├── format.ts            # форматирование денег и дат
        └── types.ts             # общие типы и метки ролей
```

## Роли

| Роль | Права |
|---|---|
| `owner` | Всё + управление командой |
| `admin` | Все операции, бюджеты, отчёты, участники |
| `manager` | Все операции и отчёты, без управления командой |
| `employee` | Только свои операции |
| `viewer` | Только просмотр |

## Данные

Суммы хранятся в **минорных единицах** (копейках) как `bigint` — без ошибок
округления. Каждый счёт в своей валюте; в отчётах пересчёт в основную валюту
команды по курсам из `fx_rates`.
